# デザイナー向けガイド：ルール（世界の法則）の作成と管理

本プロジェクトでは、ルールの定義がコードから切り離され、Unity Editor 上のアセットとして管理できるようになりました。これにより、プログラミングなしで新しいルールの追加や章（チャプター）の構成を行うことができます。

## 1. ルールの基本構成

ルールは以下の 3 種類のアセットで構成されます。

1.  **Rule Asset**: 「A ならば B」という最小単位のルール定義。
2.  **Rule Page Asset**: 章（チャプター）単位のルールリスト。
3.  **Rulebook Asset**: ゲーム全体で使用するページ（章）のリスト。

---

## 2. 新しいルールを作成する手順

### ステップ 1: Rule Asset の作成
1.  Project ウィンドウで右クリック > `Create` > `FlipLogic` > `Rulebook` > `Rule Asset` を選択します。
2.  作成されたアセットを選択し、Inspector で以下の項目を設定します。
    - **Rule Id**: プログラム内部で識別するためのID（例: `rule_fire_ice`）。
    - **Rule Name**: UI に表示されるルールの名前。
    - **Description**: ルールの詳細説明。
    - **Condition (P)**: 肯定形・否定形のテキスト（例: 「火のマスにいる」「火のマスにいない」）。
    - **Result (Q)**: 肯定形・否定形のテキスト（例: 「死ぬ」「死なない」）。
    - **Mechanics**:
        - **Subject Filter P**: ルールを適用する対象の絞り込み（例: 氷属性の敵のみ）。
        - **Tag Condition P**: 条件側のタグ判定設定（対象、キー、値）。
        - **Tag Result Q**: 結果側のタグ操作設定（対象、キー、値、操作種別、持続ターン）。

#### 💡 重要な設定項目: Target (対象)
「どのエンティティ」の「どこ」を判定・変更するかを指定します。
- **Entity**: そのエンティティ自身（例: スライム、プレイヤー）。
- **TileOfEntity**: そのエンティティが今立っている「足元のマス」。
- **Tile**: 特殊な特定マス（※現在は将来拡張用として予約されています）。

**使用例**: 「火の（TileOfEntity）マスに乗った、氷属性の（Entity）スライムは、死ぬ（Entity）」。

### ステップ 2: Rule Page (章) への登録
1.  Project ウィンドウで `Rule Page Asset` を作成（または既存のものを選択）します。
2.  `Chapter` 番号や `Title` を設定します。
3.  `Rules` リストに、ステップ 1 で作成した `Rule Asset` をドラッグ＆ドロップして追加します。

### ステップ 3: Rulebook への登録
1.  Project ウィンドウで `Rulebook Asset` を選択します。
2.  `Pages` リストに、ステップ 2 で作成した `Rule Page Asset` を追加します。

---

## 3. ゲームへの反映 (GameManager)

作成した `Rulebook Asset` をゲームに反映させるには：

1.  シーン内の `GameManagerObj` を選択します。
2.  `GameManager` コンポーネントの **Initial Rulebook** フィールドに、作成した `Rulebook Asset` をドラッグ＆ドロップします。
3.  シーンを保存します。次回起動時から、このアセットに基づいたルールが適用されます。

---

## 4. バリデーション機能（入力ミス防止）

タグの Key や Value に、`TagKeyRegistry`（タグマスタ）で定義されていない値を入力すると、**Unity の Console ウィンドウに黄色い警告 (Warning)** が表示されます。

- **例**: `Element` というキーに対して `Water` という値がマスタにない場合、「未定義のタグまたは値が指定されています」と警告が出ます。
- **対処**: 値が正しいか確認するか、必要に応じて `Assets/Resources/TagKeyRegistry.asset` に新しい定義を追加してください。

> [!TIP]
> ルールを新しく作成した際は、Play モードで実行し、Console に警告が出ていないか確認することを推奨します。
