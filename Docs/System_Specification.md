# Flip Logic システム仕様書 (Version 0.1)

## 1. 概要
『Flip Logic』は、論理学の「命題（P→Q）」を改変することで世界の法則を書き換え、有利な状況を作り出して戦う論理改変RPGです。

## 2. コアシステム
### 2.1 タグシステム
全ての `GameEntity` は `TagContainer` を持ち、複数のタグ（`TagDefinition`）を付与できます。
- **属性（Element）**: 火、氷、雷など。
- **状態（Status）**: 即死、防御中など。
- **持続時間**: ターン経過（Tick）により自動で減少・消滅します。

### 2.2 グリッド＆ターン
- **グリッド**: 1.0ユニット単位の座標系。
- **ターンマネージャー**: プレイヤーの行動（移動、スキル使用）を1ターンとしてカウントし、全エンティティのタグ持続時間を更新します。

## 3. ロジックエンジン（論理改変）
### 3.1 命題構造
ルールは 「条件(P) → 結末(Q)」 の形式で定義されます。
- **P (Condition)**: エンティティが特定のタグを持っているか（例: `Target has Tag "Ice"`）。
- **Q (Effect)**: 条件合致時に発生するタグ操作（例: `Add Tag "Dead"`）。

### 3.2 論理変換（Rule Hack）
プレイヤーは以下の操作で既存のルール `P → Q` を変形できます。
- **逆 (Converse)**: `Q → P`
- **裏 (Inverse)**: `¬P → ¬Q`
- **対偶 (Contrapositive)**: `¬Q → ¬P`

## 4. バトルシステム
### 4.1 エンカウント
フィールド上でプレイヤーと敵エンティティが隣接（上下左右）した際に遷移します。

### 4.2 コマンド
- **たたかう**: 基本攻撃。
- **ぼうぎょ**: 「防御中」タグを自身に付与。
- **ルールブック**: `RuleHackPanel` を開き、現在有効なルールを改変します。
- **にける**: 一定確率でバトルを離脱。

## 5. 実装コンポーネント
| クラス | 役割 |
|---|---|
| `BattleManager` | バトルの進行・フェーズ管理、UIへの橋渡し |
| `RuleEvaluator` | ターン毎に全エンティティに対しアクティブルールを評価・適用 |
| `GridMap` | グリッド座標とワールド座標の変換、障害物判定 |
| `TagContainer` | タグの追加・削除・値取得・持続時間管理 |

## 6. ディレクトリ構造
- `Assets/Scripts/Core`: 基底クラス、マネージャー類
- `Assets/Scripts/Data`: ScriptableObject等、データ定義
- `Assets/Scripts/Logic`: 論理評価エンジン
- `Assets/Scripts/Battle`: バトル関連UI・ロジック
- `Assets/Scripts/Tutorial`: デモ用セットアップ・生成スクリプト
