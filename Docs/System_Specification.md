# Flip Logic システム仕様書 (Version 0.1)

## 1. 概要
『Flip Logic』は、論理学の「命題（P→Q）」を改変することで世界の法則を書き換え、有利な状況を作り出して戦う論理改変RPGです。

## 2. コアシステム
### 2.1 タグシステム（不変の世界法則）
タグはエンティティ（プレイヤー・敵）だけでなく、**グリッドのマス（セル）**やステージギミックにも共通して付与される属性の最小単位です。
- **属性（Element）**: 火、氷、雷など。マスの見た目（オーバーレイ表示）にも直結します。
- **状態（Status）**: 即死、防御中、毒など。
- **絶対法則（Behavior）**: `TagBehaviorRunner` を通じて、特定のタグが付与された際やターン終了時に何を引き起こすか（効果）を定義します。
  - 例: `InstantDeath` BehaviorIdを持つタグが付与されるか、条件を満たすと、そのエンティティのHPが0になり死亡する。

### 2.2 グリッド＆ターン
- **グリッド**: 1.0ユニット単位の座標系。各マスは個別にタグを保持可能です。
- **ターンマネージャー**: プレイヤーの行動を1ターンとし、全エンティティおよび全マスのタグ状態（持続時間、効果発動）を更新します。

## 3. ロジックエンジン（可変のルールブック）
### 3.1 命題構造
ルールは 「条件(P) → 結末(Q)」 の形式で定義され、ルールブックに記載されています。これらはプレイヤーが「論理改変」可能な対象です。
- **主語フィルタ (SubjectFilterP)**: このルールがどのエンティティ群に適用されるかの前提条件（例: 「氷属性」を持つエンティティのみ）。
- **ターゲット指定**: PおよびQは、`Entity`（対象自身）、`TileOfEntity`（対象がいる場所のマス）のいずれかにスコープを設定できます。
- **P (Condition)**: 対象が特定のタグを持っているか。特例として `Status:InstantDeath` を条件とした場合、HPが0の「物理的な死亡」状態でも条件を満たしたとみなすハックが存在します。
- **Q (Effect)**: 条件合致時に発生するタグ操作（付与・除去）。`BehaviorId`を指定することでシステム側（TagBehaviorRunner）の直接的な振る舞い（即死など）を誘発します。

### 3.2 論理変換（Rule Hack）
プレイヤーはルールブック内の命題に対し、以下の操作を行えます。
- **否定 (Negate)**: PまたはQの真偽を独立して反転させる（¬P, ¬Q）。
- **入替 (Swap)**: PとQの位置を入れ替える。
- **組合せ**: これらを組み合わせることで「逆・裏・対偶」を含む自由な論理改変が可能です。

## 4. バトルシステム
### 4.1 エンカウント
フィールド上でプレイヤーと敵エンティティが隣接（上下左右）した際に遷移します。

### 4.2 コマンドとターン進行
- **たたかう**: 基本攻撃。
- **ぼうぎょ**: 「防御中」タグを自身に付与。
- **ルールブック**: `RuleHackPanel` を開き、現在有効なルールを改変します。改変後はただちに敵のターンに移行します。
- **にける**: 一定確率でバトルを離脱。

### 4.3 ターン終了フェーズ (TurnEnd)
探索時と同様に、バトル中も敵の行動後に `TurnEnd` フェーズが挟まれます。
- **ルールの評価**: `RuleEvaluator` が全エンティティに対してルール評価を行います。
- **死因の表示**: ルールの結果によってエンティティが死亡した場合、「ルールの効果『氷スライムは死ぬ』により、氷スライムは死んだ！」のように、ハードコードを避けた動的で自然な死因メッセージが表示されます。チュートリアルでルール改変によって死亡を回避した場合は「生き延びた！」とアナウンスされます。

## 5. 実装コンポーネント
| クラス | 役割 |
|---|---|
| `BattleManager` | バトルの進行・フェーズ管理、UIへの橋渡し |
| `RuleEvaluator` | ターン毎に全エンティティに対しアクティブルールを評価・適用 |
| `GridMap` | グリッド座標とワールド座標の変換、障害物判定 |
| `TagContainer` | タグの追加・削除・値取得・持続時間管理 |

## 6. ディレクトリ構造
- `Assets/Scripts/Core`: 基底クラス、マネージャー類
- `Assets/Scripts/Data`: ScriptableObject等、データ定義
- `Assets/Scripts/Logic`: 論理評価エンジン
- `Assets/Scripts/Battle`: バトル関連UI・ロジック
- `Assets/Scripts/Tutorial`: デモ用セットアップ・イベントシナリオスクリプト

## 7. チュートリアルの実装仕様と演出
チュートリアルは、プレイヤーに「世界のルール」と「その改変の力」を段階的に理解させるためのシークエンスとして実装されています。

### 7.1 シナリオ駆動型イベントシステム
- `ScenarioRunner` と `ScenarioStep` クラスを用いて、メッセージの表示、クリック待機、エンティティの強制移動などをステップバイステップで実行します。
- これにより、意図しないタイミングでのメッセージスキップや、敵の自律敵なランダム移動によるシナリオ崩壊を防ぎます（チュートリアル用敵は `Stationary` AIで固定化）。

### 7.2 世界の法則の直観的なデモ
1. **ルールの提示**: 最初の氷スライムが、シナリオの制御によって強制的に「火のマス」の上へと移動させられます。
2. **評価と死**: ターン処理が走り、初期ルールである「氷スライムが火のマスにいる → 氷スライムは死ぬ」が適用され、プレイヤーの目の前でスライムが即死・消滅します。
3. **プレイヤーへのバトンタッチ**: 第2の氷スライムが出現し、エンカウント（バトル開始）します。このバトル内でプレイヤーは「ルールブック」を開き、自らの手で死を回避するか、逆に条件を書き換える（例: 死んだら火のマスになる）等して、システムの自由度を体験します。内部ロジックの判定結果により、バトル内で死亡の有無とその理由が動的にメッセージとして表示されます。
